# Вопросы для самопроверки

# 1. Зачем нужны `variables` в GraphQL запросе? Почему нельзя просто склеивать строки?
Переменные позволяют отделить структуру запроса от данных, что повышает безопасность (предотвращает GraphQL-инъекции), улучшает кэширование на стороне сервера (запрос парсится один раз) и делает код чище и поддерживаемее.

# 2. Из каких полей состоит стандартный JSON-запрос к GraphQL серверу?
Стандартный запрос содержит как минимум поле "query" со строкой запроса. Опционально может включать поле "variables" со словарём переменных и поле "operationName" для указания имени операции, если в запросе их несколько.

# 3. Почему у GraphQL обычно только один URL endpoint?
GraphQL использует единую точку входа, потому что вся логика работы с данными описывается в самом запросе (схема и резолверы). Это упрощает архитектуру и избавляет от множества эндпоинтов как в REST.

# 4. Что означает наличие поля `errors` в ответе сервера? Может ли оно прийти вместе с `data`?
Поле errors сигнализирует о том, что при выполнении запроса произошли ошибки. Оно может присутствовать вместе с data — это означает частичный успех: часть данных получена, а часть запросов не удалась (например, из-за проблем с доступом или несуществующих полей).

# 5. Как клиент должен обрабатывать ошибки, если HTTP статус ответа 200?
Даже при HTTP 200 необходимо проверить наличие поля errors в теле ответа. Если оно есть, следует обработать ошибки согласно логике приложения (логировать, показывать пользователю и т.д.), так как GraphQL использует статус 200 для успешной доставки ответа, а ошибки уровня приложения передаются внутри JSON.

# 6. В чем плюсы использования типизированных клиентов (например, Apollo Client или Relay) перед простыми HTTP-запросами?
Типизированные клиенты автоматически генерируют TypeScript-типы на основе схемы, упрощают управление кэшем, поддерживают optimistic updates, подписки через WebSocket и декларативную загрузку данных. Это снижает количество шаблонного кода и уменьшает вероятность ошибок несоответствия типов.
