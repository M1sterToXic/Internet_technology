# Вопросы для самопроверки

1. В чем главное концептуальное отличие GraphQL от REST?
В REST клиент получает предопределённые наборы данных с фиксированных эндпоинтов, а сервер диктует структуру ответа. В GraphQL клиент сам описывает в запросе, какие поля ему нужны, и получает ровно их. Контроль над данными смещается на сторону клиента.

2. Что такое Schema Definition Language (SDL)?
SDL — это синтаксис для описания типов, запросов и мутаций в GraphQL. Он позволяет задать контракт между клиентом и сервером в виде, понятном обеим сторонам.

3. Чем Query отличается от Mutation? Могут ли они выполняться параллельно?
Query предназначен для чтения данных (идемпотентные операции, без побочных эффектов).

Mutation — для изменения данных (создание, обновление, удаление).
Мутации выполняются строго последовательно, чтобы избежать состояний гонки. Запросы (Query) могут выполняться параллельно, так как они не меняют состояние.

4. Что такое "резолвер" (resolver) и кто его вызывает?
Резолвер — это функция, которая знает, как получить значение для конкретного поля схемы (например, из базы данных, другого API или вычислений). GraphQL-сервер вызывает резолверы по мере обхода запроса, передавая им родительский объект, аргументы и контекст.

5. Как решается проблема N+1 запросов в GraphQL (dataloader)?
Проблема N+1 возникает, когда для каждого элемента списка выполняется отдельный запрос к БД. Решение — использование DataLoader: механизма, который батчит запросы и кэширует результаты на время одного HTTP-запроса. Например, вместо N запросов findUser(id) выполняется один findUsers(ids)
6. Почему в GraphQL сложно реализовать кэширование на уровне HTTP (как в REST)?

В REST каждый эндпоинт соответствует конкретному ресурсу, и можно кэшировать ответ по URL. В GraphQL все запросы идут на один эндпоинт, тело запроса (query) динамическое, поэтому стандартное HTTP-кэширование по URL не работает. Требуются более сложные подходы: кэширование на уровне полей, персистентные запросы или использование сервисов типа Apollo Client с нормализованным кэшем.
