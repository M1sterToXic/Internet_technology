# Вопросы для самопроверки

## 1. Что такое Server Streaming?

Server Streaming — это когда клиент отправляет один запрос, а сервер отвечает потоком данных. Соединение остаётся открытым пока сервер шлёт сообщения.

**Примеры:** лента новостей, биржевые котировки, стриминг логов.

## 2. Чем Server Streaming отличается от List?

- **Streaming** — данные приходят порциями, можно обрабатывать сразу
- **List** — клиент ждёт весь ответ целиком

## 3. Какие метрики производительности существуют?

- **Latency** — задержка (время от запроса до ответа)
- **Throughput** — пропускная способность (RPS, запросов в секунду)
- **CPU Usage** — загрузка процессора

## 4. Почему важно переиспользовать канал (Channel)?

Создание нового канала — дорогая операция (TCP соединение, handshake). При переиспользовании все запросы идут через одно соединение — это быстрее.

## 5. Когда REST удобнее gRPC?

- Простые CRUD операции
- Нужна простая отладка (JSON читается легче)
- Публичные API
- Браузерные клиенты

## 6. Что такое Head-of-Line Blocking?

Это когда следующий запрос ждёт пока завершится предыдущий. **HTTP/2 решает** мультиплексированием — несколько запросов идут одновременно в одном соединении.
