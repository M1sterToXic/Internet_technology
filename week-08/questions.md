# Вопросы для самопроверки

# 1. Зачем нужен Signaling сервер, если WebRTC — это P2P технология?
Signaling сервер нужен для начального обмена служебной информацией между клиентами (SDP-описания и ICE-кандидаты). Браузеры не могут соединиться напрямую, пока не узнают параметры друг друга. После обмена данными Signaling сервер больше не участвует — трафик идёт напрямую между клиентами (P2P).

# 2. Что такое NAT и почему он мешает прямому соединению?
NAT (Network Address Translation) — это технология, которая позволяет нескольким устройствам в локальной сети использовать один внешний IP-адрес. NAT мешает прямому соединению, потому что устройства внутри сети имеют приватные IP-адреса (например, 192.168.x.x), которые не видны извне. Внешний клиент не знает, как «достучаться» до конкретного устройства за NAT.

# 3. В чем разница между STUN и TURN серверами?
STUN сервер помогает клиенту узнать свой внешний IP-адрес и порт (публичный адрес, который видит интернет). Это нужно для установки P2P соединения. TURN сервер — это запасной вариант: если P2P соединение не удалось (строгий NAT, файрволы), трафик идёт через TURN сервер как через посредника. TURN требует больше ресурсов и обычно платный.

# 4. Что такое ICE Candidate?
ICE Candidate — это один из возможных сетевых адресов клиента (IP:порт), через который можно установить соединение. Клиент собирает несколько кандидатов: локальный адрес, внешний адрес (через STUN), адреса TURN. Затем отправляет их собеседнику через Signaling сервер. Собеседник пробует подключиться по каждому кандидату, пока не удастся установить соединение.

# 5. Почему WebRTC обычно использует UDP, а не TCP?
WebRTC использует UDP, потому что для аудио и видео важнее низкая задержка, чем гарантированная доставка каждого пакета. Если пакет потерялся — проще пропустить кадр, чем ждать его повторной отправки (как в TCP). UDP также позволяет отправлять данные без установки долгого соединения и без overhead на подтверждение каждого пакета.

# 6. Что такое SDP (Session Description Protocol) и какая информация в нем содержится?
SDP — это текстовый формат описания параметров медиа-сессии. В SDP содержится: какие кодеки поддерживаются (видео: H.264, VP8; аудио: Opus, PCMU), разрешение видео, частота дискретизации, ICE-кандидаты, DTLS-ключи для шифрования. SDP не передаёт медиа — только описывает, как клиенты могут договориться о параметрах соединения.
