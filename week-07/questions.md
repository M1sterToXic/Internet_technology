# Вопросы для самопроверки

## 1. Зачем нужен файл `.proto`?

Файл `.proto` описывает структуру данных и сервисы. Из него генерируется код для Python и других языков. Это контракт между клиентом и сервером.

## 2. Что такое Unary RPC?

Unary RPC — это обычный запрос-ответ. Клиент отправляет один запрос, сервер возвращает один ответ. Похож на REST GET/POST запрос.

## 3. Почему gRPC эффективнее REST?

- **Бинарный формат** — меньше размер сообщений чем JSON
- **Быстрее парсинг** — бинарные данные обрабатываются быстрее текста
- **HTTP/2** — одно соединение на все запросы

## 4. Что такое обратная совместимость в Protobuf?

Можно добавлять новые поля, старые клиенты их игнорируют. **Нельзя менять номера полей** — это сломает совместимость.

## 5. Что такое Stub и Servicer?

- **Stub** — клиентский класс для вызова RPC
- **Servicer** — серверный класс, в котором вы реализуете логику

## 6. Когда REST, а когда gRPC?

- **REST** — публичные API, простые CRUD, браузеры
- **gRPC** — микросервисы, высокая производительность, streaming
